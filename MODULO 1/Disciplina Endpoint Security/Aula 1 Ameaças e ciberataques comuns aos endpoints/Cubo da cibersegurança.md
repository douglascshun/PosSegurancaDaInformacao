# Cubo da cibersegurança

`A ideia desse conceito é desenvolver sistemas com garantia de informações e como as informações estão interligadas.`

![[O cubo da cibersegurança.png]]

Esse cubo tem 9 formações a primeira base desse cubo é uma das mais importantes na área da segurança da informação chamada de Triad:

![[Triad.png]]

`Confidencialidade`: Sistema confiável onde só eu consiga acessar ele.

`Integridade`: A informação deve chegar integra e completa. 

`Disponibilidade`: A informação deve estar disponível para eu acessa-la quando solicitado.

# Segunda dimensão do cubo 

- `Dados inativos ou armazenados`

- `Dados em trânsito`

- `Dados de processamento`

`Dados inativos`: Dados inativos são dados como currículos, que a empresa pode guardar durante dois anos, caso precise contratar alguém, ele não é consultado constantemente no sistema, mas permanece armazenado.

`Dados armazenados`: Dados usados constantemente, como os dados de consultas de cadastros de catracas, onde na recepção você apresenta algo e é feita uma consulta sobre seu registro para liberar sua entrada.

`Dados em trânsito`: Dados em trânsito pode ser o envio de uma mensagem enviada via WhatsApp, onde a mensagem é transitada de um dispositivo até o outro de meio seguro. 

`Dados em processamento`: Dados em processamento são dados que estão constantemente rodando, como os dados de planilhas que estou mexendo.

# Terceira dimensão do cubo

- `Tecnologia`
- `Políticas e práticas`
- `Fatores humanos`

`Tecnologia`: Qual tecnologia vou utilizar, Windows, Linux, Python.

`Políticas e práticas`: A empresa que trabalho,ela tem uma politica boa de backup, se acontecer um ataque, eu tenho uma politica para recuperar esses dados?

`Fatores humanos`: Uso errado de credencial, tirar foto de crachá e postar em rede social, compartilhar senhas.
