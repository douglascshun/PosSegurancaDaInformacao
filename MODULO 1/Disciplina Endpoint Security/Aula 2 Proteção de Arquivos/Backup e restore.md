## Backup e restore

`Backup`: Cópia de segurança.

`Restore`: Restauração de cópia de segurança.

Fazer um backup dos seus dados, significa ter guardado uma cópia exata deles em algum outro lugar.

Os serviços de Backup podem guardar essa cópia em nuvem (sistemas de armazenamento de dados da internet) ou em outro dispositivo de armazenamento.

# Políticas de backup

- `A frequência de realização dos backups`
(dia, semanal, mensal)

- `O tipo de backup`
(Completo, toda informação SO. Incremental, só que o foi acrescentado naquele dia eu vou fazer o backup. Diferencial, só salva o que é diferente. De arquivo, onde salvo só os arquivos, não o SO. Imagem, copio uma imagem do servidor ou do banco de dados e salvo. Backup na nuvem, salvar em uma plataforma ou na rede)

- `Quais dados precisam ser armazenados`
(Dados quentes, dados que são valiosos)

- `Onde serão guardadas as cópias`
(servidor remoto, nuvem, outra filial, HD externo)

- Os funcionários responsáveis pela atividade.
(Somente pessoas de extrema confiança)
